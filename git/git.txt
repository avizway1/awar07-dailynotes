Branching helps us to create a seperate line of development to work on new features, bug fixes, enhancements without touching your main branch/codebase.

git branch <branch-name>	--> create branch
git switch <branch-name>	--> switching to the branch

git branch -b <branch-name>	--> create branch and switch automatically -- this command is not working
git switch c <branch-name>      -- Create branch and switch automatically 
---

Scenario: 
branches edit diff files --> merge succesful automatically
branches edit same file but diff lines --> merge succesful automatically
Branches edit same file, same line --> Conflict (Resolve manually)

--> Thumb Rule : We dont work on Main / Master branch.. Mostly Devs work on "feature/BugFix/Hotfix"

Thumb Rule : do not push/commit unfinished work.


We can store unfinished changes into a secure locaqtion, We called it as stash


git stash						--> it stash current changes
git stash push -m "message"		--> it stash current changes with messages
git stash list					--> list all stashes
git stash apply					--> apply latest stash
git stash pop					--> apply the stash and deletes the stash
git stash show -p				--> shows the stashed content
git stash clear					--> clears the stash
