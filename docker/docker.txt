Restart Policies: 

For production workloads, we use "always" most of the times.

no				- never restart (default)
on-failure		- Restart only if container exits with error
always			- Always restart, including on host reboot


docker run -d --restart=always -p 8080:80 nginx:1.28

Network Modes in Containers:


bridge : Default network mode. Containers gets their own virtual network.
host : Container shares host machine's network and Port directly.
none : No network at all.
container : Shares network with another container. 

docker network ls

docker network inspect bridge

docker network create mynetwork

docker run -dit --name c1 --network mynetwork alpine
docker run -dit --name c2 --network mynetwork alpine


---

docker run -d --network host nginx


---

docker run -it --network none alpine sh

-------

We can add an existing container to a network:

docker network connect <net-name> <cotnainer-name>


docker network connect mynetwork 14e46f0075d5

---


bridge : 

own virtual IP inside docker network
Required port mapping
This is an isolated network
Slightly slower as additional hop added
Best for Multi Container applications.

Host: 
Shares host's IP and port
Port mapping not required. Uses host ports.
Less isolated, as it uses host network
Bit faster as it uses host networtk directly
system level tooling, Host monitoring agents

Named Volume :

Anonymous Volume: Automatically one volume creates with random name

---


BIND MOUNT : Pick a Host directory and mount it to container.

docker run -d -p 80:80 -v /home/ec2-user/shared-data:/usr/share/nginx/html nginx:1.28

=====

if you want to make any volume readonly, we need to pass addl parameter.

docker run -d -p 80:80 -v mydata:/usr/share/nginx/html:ro nginx:1.28

DOCKERFILE: 


FROM 		: sets the base image. THis should be first instruction in every docker file. 
FROM ubuntu:20.5
FROM python:3.12-slim
FROM alpine:latest

RUN 		: Executed comamnds during the build process. 
RUN apt-get update && apt-get install -y nginx
RUN adduser appuser

WORKDIR		: We can set the working directory for all subsequent instruction.
WORKDIR /app
WORKDIR /usr/share/nginx/html

COPY		: Copies the files/Directories from our host machiene into the image. (use most of the times)
COPY /home/ec2-user/ /app/
COPY . .

ADD			: THis is similar to COPY, but This supports (zip auto extracting and download remote url)
ADD mydata.tar.gz /app/

---
CMD			: This sets the "default commnd" to run when the container starts. This "gets overriden" if you pass arguments when running "docker run " command.
CMD ["nginx","-g", "daemon off;"]

ENTRYPOINT	: Defines a command that "always runs", even if you pass any arguments wile runnning container with "docker run"
ENTRYPOINT ["python", "app.py"]

ENTRYPOINT ["python", "app.py"]   # Always runs
CMD ["--port", "8080"]            # Default args, user can override
---

ENV			: Set Environment variables
ENV	APP_ENV=Production
ENV COSTID=AAZAA

EXPOSE		: We can mention the container listerns on which particular port. (We always need to pass -p argument while running the container)
EXPOSE 80

VOLUME 		: Create a mount point for persintent or shared data/storage.
VOLUME /var/lib/mysql

USER		: sets the user for running the container. ***We never run container with root user for better security.
USER appuser

LABEL		: We can add any metadata to the container
LABEL owner="avinash@aviz.com" Version="2.0"

HEALTHCHECK	: tells docker to check healthcheck based on the arguments we pass along with healthcheck.

HEALTHCHECK CMD curl --fail http://localhost:80 || exit 1



While SIGTERM is the standard for most applications to initiate a "polite" exit, some specific software uses SIGQUIT to trigger a graceful shutdown that finishes active tasks before terminating.

Docker compose helps us to run multiple containers at a time using a single YAML file.file name  "docker-compose.yaml"

Install Docker Compose with below command

# Ensure the directory exists
mkdir -p ~/.docker/cli-plugins

# Download the correct binary (copy the entire line)
curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-$(uname -m) -o ~/.docker/cli-plugins/docker-compose

# Make the plugin executable
chmod +x ~/.docker/cli-plugins/docker-compose
